Frameworks PHP

Qu'est-ce qu'un framework ?

Frame : Cadre
Work : Travail

Ensemble de composants structurés permettant de mettre en place l'architecture d'un logiciel

Valable dans la majorité des domaines y comprit les sites

Une librairie est un outil ou un ensemble d'outils
Un framework est un établi avec déjà plusieurs outils

Lorsque vous développez votre structure avec vos outils, vous créez un genre de framework

Un framework n'est pas un CMS

Un CMS peut être basé sur un framework

Exemples :
Drupal 8 avec Symfony
Uzik CMS avec Zend

Intérêts

  • Gain de temps / productivité
  • Moins de connaissances requises
  • Moins d'erreurs
  • Travail en groupe sur une base connue des membres
  • Communauté
  • Documentation

Contreparties

  • Potentiellement "lourd"
  • Méconnaissance du fonctionnement du Framework
  • Complexité (entre développeur de différents niveaux)
  • Mises à jour
  • Courbe d'apprentissage

Quelques framesworks PHP

Il existe de nombreux Frameworks PHP

Il en existe pour les autres languages

La majorité des Framework récents nécessitent PHP 5.3

Structure souvent MVC (?)

Symfony

http://symfony.com

  • Version actuelle : 2.4.4
  • Idéal pour les sites : Moyens et gros
  • Courbe d'apprentissage : Lente
  • Utilisation : Très répandue
  • Charge : Lourde

  • Idéal pour les sites : Moyens et gros

Il est important de bien choisir le Framework selon le besoin

Une conversion vers un autre Framework est souvent impossible

Un Framework "lourd" nécessitera un serveur plus puissant et donc plus coûteux

Certains hébergeurs peuvent être incompatibles (version de PHP)
